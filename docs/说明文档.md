# 遥操作说明文档

## 1. 基本说明

整体遥操作项目需要配置的主要有两个部分，即RoboTele和televuer。其次的话还有一个基于ROS2开发的可视化项目HumanoidViz，具体详见 https://github.com/Fa1lin9/HumanoidViz 。

其中RoboTele是cpp项目，而televuer是python项目。且televuer项目在RoboTele的utils目录下。

此外,utils目录下还有urdfly这一项目。这个项目也是开源项目，用来看URDF模型的，具体可以在github里搜索。

## 2. 安装说明

RoboTele和televuer都有对应的安装文档，这里不过多赘述。HumanoidViz是ROS2的package，没其他的依赖，常规安装即可。

## 3. 使用说明

在使用之前先阐明一下各个模块的基本作用和流程：televuer实现XR设备数据的读取和转发；RoboTele实现数据的读取和解算，最后通过DDS库发布在ROS2的话题中。

其中RoboTele运行的程序是MainDemo这个可执行程序。这里建议使用Qt Creator这个IDE，用起来还是比较方便的。如果不会用的话可以去查一下教程，还是挺简单的。使用的时候直接选择要执行的程序然后执行即可。

此外，配置文件放在RoboTele目录下的config文件夹，一般来说只用修改config/Teleoperate/目录下各个机器人的相关JSON文件。其他子文件夹里的配置文件我都基本调好了，不用修改。要改的话也只是改config/Teleoperate/文件夹里具体机器人JSON文件的FPS和控制使能部分。

需要注意的是，一般来说优化求解消耗的时间是10ms上下，所以帧率FPS可以根据这个来调整，目前给的是25。而使能模块指的是下面这个部分，主要是觉得要不要控制头、双手和双臂。
```json
    "BodyEnable": {
        "Head":         true,
        "Waist":        false,
        "LeftArm":      true,
        "RightArm":     true,
        "LeftLeg":      false,
        "RightLeg":     false,

        "LeftHand":     true,
        "RightHand":    true
    },
```

另外，遥操作的启动和结束手势是按照特定操作员进行设置的，也没有通过JSON文件来进行配置。所以如果需要修改的话还是需要自己阅读源码来修改，具体是在HandGestureDetector中。

## 4. 使用流程

一般是先开RoboTele里的MainDemo,然后打开televuer里的send_online_data，最后使用Apple Vision Pro进入相关界面。如果不懂Apple Vision Pro怎么进入相关界面的话可以去看宇树的遥操作项目xr_teleoperate。
